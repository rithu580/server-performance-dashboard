# Virtual Server Monitoring & Performance Dashboard

## Project Overview

This project analyzes the performance and health of virtual servers using **Power BI**. The dashboard visualizes key metrics such as CPU utilization, memory usage, network traffic, server availability, and downtime to help monitor infrastructure performance and quickly identify potential issues.

## Objectives

* Monitor server resource usage in real time.
* Identify servers with high CPU utilization.
* Analyze memory usage across servers.
* Detect unusual network traffic patterns.
* Evaluate system reliability through uptime and downtime analysis.

## Dashboard Visualizations

The Power BI dashboard includes the following key visuals:

1. **CPU Utilization Trend (Line Chart)** – Displays CPU usage over time to identify spikes and performance fluctuations.
2. **Top Servers by Memory Usage (Bar Chart)** – Shows servers with the highest memory consumption.
3. **Network Traffic Trend (Area Chart)** – Monitors network activity and detects unusual traffic patterns.
4. **Server Availability KPI (Card Visual)** – Displays the average availability of servers.
5. **High CPU Alert Table** – Lists servers exceeding the CPU utilization threshold.
6. **Downtime vs Uptime (Donut Chart)** – Visualizes system reliability by comparing uptime and downtime.

## Tools Used

* **Power BI Desktop** – Data visualization and dashboard development
* **GitHub** – Version control and project hosting

## Outcome

The dashboard provides a centralized view of server performance, helping administrators quickly identify performance issues and maintain system reliability.
